# 🚀 Brain Agent 2.0: The `.specs/` Architecture Era

Welcome to **Brain Agent v2.0.0**! This is our biggest (and best) structural update yet. We have retired the old loose, `Brain/` folder-based approach in favor of a **new professional, predictive, and modular architecture**, driven by the `.specs/` folder and the concept of the *Socratic Gate*.

This version was redesigned not just for code, but in a **didactic and engineered way for maximum reproducibility** of your Artificial Intelligence (Code Agents).

---

## 🎯 Why did we change? The Previous Problem

Up until v1.x, agents depended on the `Brain/` folder. However:
1. **Redundancy:** The agent would get lost trying to keep multiple documentations synchronized, causing hallucinations.
2. **Context Bloat:** Many long and irrelevant files were read unnecessarily.
3. **Blind Coding:** Often the agent would start coding before even properly mapping the scope or planning the side effects.

---

## ✨ The New Architecture (What's new?)

We solved this pain for good by introducing three fundamental pillars to the repository:

### 1. The System of Truth: The `.specs/` folder 🗂️
Now all your project management is firmly structured in `.specs/`. It acts as the primary source of truth. If the chat conflicts with the `.specs/` files, the AI will always respect the documentation.

Standardized main files:
* `@STACK` (`.specs/project/STACK.md`): Defines your entire tech ecosystem.
* `@ARCHITECTURE` (`.specs/project/ARCHITECTURE.md`): Your tech bible – design patterns, models, routes.
* `@STATE` (`.specs/project/STATE.md`): Your "daily captain". Lists blockers, architectural decisions, and recently learned lessons.
* `@TASKS` (`.specs/features/*/tasks.md`): The microscopic "to-do list" the AI works on, checking step-by-step before committing or reporting as done.

### 2. The Socratic Gate 🏛️
The agent **NO LONGER WRITES CODE BLINDLY**. We implemented a master orchestrator.
Whenever a medium/high complexity order is given ("Change the layout", "Refactor the backend"), the agent must stop and process logical steps:
1. Validate if it violates the current architecture (@ARCHITECTURE).
2. Load the required skill.
3. Confirm if the documentation needs updating before taking action.

### 3. Dynamic AI Skills (`.agents/skills/`) 🧩
Specific skills (like `code-review`, `debugging`, `testing`, `brain`/planning) are now isolated in `.agents/skills/`. The repository acts as an operating system that injects the skill at the exact right moment, drastically reducing costs and token usage in the base prompt, while enhancing review and delivery quality.

---

## 🚀 Practical Impact

By integrating the `v2.0.0` model into your workflow:
* Your **refactors** won't break the application without warning you about side effects and dependencies.
* It will know **exactly** where your routes, DB, and controllers are.
* The documentation will evolve **on its own**, side by side with your `commits`.
* The **Install Script is now Non-Destructive**. You can run the setup over existing projects (even those with native `.agents/` folders) and it will securely inject the Brain architecture while preserving your existing rules.

**Goodbye complexity and hallucinations, hello absolute Tech Lead-level efficiency.** Enjoy Brain Agent 2.0!
